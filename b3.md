# Bài 3: Đọc hiểu & Dò lỗi qua Prompt

## Vì sao prompt thô dễ bỏ sót lỗi logic

Prompt "Mã này bị lỗi gì?" quá ngắn nên AI thường chỉ kiểm tra lỗi cú pháp, lỗi biên dịch hoặc lỗi runtime dễ thấy. Trong bài này, code **không sai cú pháp** nên AI rất dễ bỏ qua lỗi logic lặp bên trong vòng `for`.

Lỗi thật sự nằm ở chỗ `j` bắt đầu từ `i`, khiến `arr[i]` bị so sánh với chính nó ngay từ đầu. Vì `arr[i] == arr[i]` luôn đúng, hàm sẽ trả về phần tử đầu tiên dù mảng không có phần tử trùng lặp thật.

Ngoài ra, prompt thô không cho test case cụ thể nên AI khó tự chứng minh lỗi bằng ví dụ thực tế. Khi không có dữ kiện kiểm thử, AI dễ trả lời chung chung và bỏ sót bug logic biên.

## Prompt tối ưu mới

~~~text
Hãy đóng vai trò là Code Auditor chuyên kiểm tra lỗi logic trong Java.

Tôi có đoạn code sau:
```java
public class DuplicateFinder {
    // Lỗi logic: j bắt đầu từ i dẫn đến việc phần tử tự so sánh với chính nó
    public static Integer findDuplicate(int[] arr) {
        for (int i = 0; i < arr.length; i++) {
            for (int j = i; j < arr.length; j++) {
                if (arr[i] == arr[j]) {
                    return arr[i];
                }
            }
        }
        return null;
    }
}
~~~

Test case cần kiểm tra:
- Input: {1, 2, 3, 4}
- Kết quả mong đợi: null vì không có phần tử trùng lặp
- Kết quả thực tế hiện tại: trả về 1

Yêu cầu:
1. Phân tích rõ vì sao code hiện tại bị sai logic.
2. Chỉ ra vị trí gây lỗi.
3. Sửa lại mã nguồn bằng cách sử dụng HashSet.
4. Đảm bảo độ phức tạp thời gian tốt hơn, từ O(N2) xuống O(N).
5. Trả về code Java hoàn chỉnh, dễ hiểu và đúng nghiệp vụ.
```

## Mã nguồn Java đã sửa bằng HashSet

```java
import java.util.HashSet;
import java.util.Set;

public class DuplicateFinder {
    public static Integer findDuplicate(int[] arr) {
        if (arr == null || arr.length == 0) {
            return null;
        }

        Set<Integer> seen = new HashSet<>();

        for (int value : arr) {
            if (seen.contains(value)) {
                return value;
            }
            seen.add(value);
        }

        return null;
    }
}
```
